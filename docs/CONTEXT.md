# Glossary

Canonical terms for the sound matching evaluation framework. This file defines language
only — implementation decisions live in `DECISIONS.md`, architecture in `PROJECT_CONTEXT.md`.

| Term | Definition |
|---|---|
| **Sound matching** | Predicting the parameter settings of a target synthesizer such that its rendered audio matches a target waveform. Synonym in the literature: inverse synthesis, synthesizer inversion. |
| **Patch** | A complete parameter configuration for a synthesizer. Synonym: preset — the two terms are used interchangeably in this project. |
| **Preset** | Same as patch. |
| **Parameter name** | The plugin-reported string identifying a parameter (e.g. `ALGORITHM`, `OP1 OUTPUT LEVEL`). The only way parameters are referred to above the wrapper. |
| **Synth-side dict** | A `dict[str, float]` mapping parameter names to normalized floats in [0,1] — the format the synthesizer wrapper accepts and returns. Categorical values are grid points on [0,1]. |
| **ML-side vector** | A flat `np.ndarray` consumed/produced by models: continuous params as floats, categorical params expanded to one-hot blocks. |
| **ParamSpace** | The abstraction owning the ordered parameter subset definition and the two-way conversion between synth-side dict and ML-side vector. |
| **Subset** | The chosen set of parameters that models estimate. All non-subset parameters are *locked* at defaults. All benchmarked models target the same subset. |
| **Locked parameter** | A parameter held at its default value and never estimated, randomized, or exposed in the dataset. |
| **Excluded parameter** | A VST-level parameter (e.g. `Bypass`, `Program`, MIDI CC passthroughs) that the wrapper never exposes at all. Stronger than locked. |
| **Render contract** | The guarantee that rendering the same synth-side dict always yields bit-identical audio. Upheld by re-applying parameters before every render. |
| **Render duration** | Total length of rendered audio (4.0 s). |
| **Note duration** | Time from note-on to note-off within the render (3.0 s); the remainder captures the release tail. |
| **Synthetic dataset** | Training data generated by randomly sampling the subset and rendering each patch. |
| **Human preset test set** | Held-out evaluation set of human-curated presets (e.g. DX7 SysEx cartridges), testing generalization to musically realistic sounds. |
| **Model family** | One of the six SLR taxonomy categories (evolutionary, CNN, transformer, VAE, flow, neural proxy/RL); the benchmark implements one representative per family. |
| **Metric panel** | The fixed set of evaluation metrics computed for every model on every test sample (parameter-side diagnostics + audio-side spectral/perceptual distances). |
| **Re-render** | Rendering a model's predicted synth-side dict through the same wrapper used for dataset generation, to compare against the target audio. |
